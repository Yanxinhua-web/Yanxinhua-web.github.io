# 什么是路由？

  • 路由是一个网络工程里面的术语
  • 路由就是通过互联网的网络把信息从源地址传输到目的地地址的活动
  • 生活中我们有没有听说过路由的概念呢？路由器
  • 路由器提供了两种机制：路由和转送
  1. 路由是决定数据包从来源到目的地的路径
  2. 转送将输入端的数据转移到合适的输出器
  • 路由中有一个非常重要的概念叫路由表
  • 路由表本质上就是一个映射表。决定了数据包的指向

# url的哈希模式跟html5的history

  • url的哈希模式：通过location.hash="aaa"设置，发现页面没有刷新，但是地址已经发生了变化
  • html5的history：
  1. history.pushState({},'','home') 此时页面地址也变化了，但是页面同样没有刷新，栈结构（先进后出）aa->bb->cc history.back()，此时页面地址是bb
  2. history.replaceState({},'','about'); 替换url，不是压栈出栈，不能返回
  3. history.go(-1) 返回上个页面等同于history.back 也可以history.go(-2)
  4. history.forward()等同于go(-1)

# 认识vuerouter

  • vuerouter是vuejs官方的路由插件，他和vuejs是深度集成的，适合用于构建单页面应用
  • 我们可以访问官网进行学习，https://router.vuejs.org/zh/
  • vuerouter是基于路由和组件的
    1. 路由用于设定访问路径，将路径和组件映射起来
    2. 在vuerouter的单页面应用中，页面路径的改变就是组件的切换

# 使用vue-router的步骤

  • 第一步：创建路由组件
  • 第二步：配置路由映射，组件和路径映射关系

    //配置路由相关的信息

import VueRouter from "vue-router";

//导入组件

import Home from "../views/Home.vue";
import About from "../views/About.vue";



//1.通过vue.use(插件)，安装插件
Vue.use(VueRouter);
//2.创建vuerouter对象
const routes = [{
    path: "/home",
    name: "Home",
    component: Home
  },
  {
    path: "/about",
    name: "About",
    component: About
  }
];

const router = new VueRouter({
  //配置路由和组件之间的应用关系
  routes
});

//导出router
export default router;

# 通过代码控制路由跳转

<template>
  <div id="app">
    <div id="nav">
      <!-- router-link全局注册的组件 -->
      <!-- <router-link to="/home" tag="button" repalce active-class="abc">首页</router-link>|
      <router-link to="/about" tag="button">关于</router-link> -->
      <button @click="homeclick">home</button>
      <button @click="aboutclick">about</button>
    </div>
    <!-- 决定渲染出来的组件在什么位置 -->
    <router-view />
  </div>
</template>

<script>
export default {
  name: "app",
  methods: {
    homeclick() {
      this.$router.push("./home");
    },
    aboutclick() {
      this.$router.replace("./about"); //不返回
    }
  }
};
</script>

//index.js
// meta 元数据 描述数据
  {
    path: "/profile",
    name: "Profile",
    component: Profile,
    meta: {
      title: "档案"
    }
  }


//index.js
//前置钩子 
router.beforeEach((to, from, next) => {
  //从from到to
  // document.title = to.matched[0].meta.title;
  document.title = to.meta.title;
  console.log(to);
  next();
})
//后置守卫 跳转后的
  router.afterEach((to, from, next) => {
    next();
  })

    name: "Profile",

    meta: {
      title: "档案"
    },
    beforeEach: (to, from, next) => {
      next()
    }

# keepalive （保持活着）

  • keep-alive是vue内置的组件，可以使被包含的组件保留状态，或避免重新渲染
  • router-view也是一个组件，如果直接被包含在keep-alive里面，所有路径匹配的视图组件都会被缓存
  //keep-alive用法
  <keep-alive>
    <router-view />
  </keep-alive>
  //这两个函数，只有该组件被保持了状态，使用了keep-alive时才有效
  activated() {},
  deactivated() {}
  //路由内守卫 记录离开时的path
  beforerouterleave(to,from,next){
    this.path=this.$touter.path;
    next();
  }

# keep-alive的属性

  • include 字符串或者正则表达，只有匹配的组件会被缓存
  • exclude 字符串或者正则表达，只有匹配的组件不会被缓存
 //用法(多个情况下格式如下，不要加空格) 排除档案页面 会被销毁，创建
  <keep-alive exclude="profile,User">
        <router-view />
  </keep-alive>
